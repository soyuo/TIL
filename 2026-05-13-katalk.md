# KakaoTalk `26.1.2` Android APK Analysis
> 대상: Android `26.1.2` / `xapk` (예: APKPure)

## 1. 분석 환경
- `jadx 1.5.5`
- `kakaotalk 26.1.2 xapk`

---

## 2. jadx에서 기본 진입
1. jadx 실행 후 **파일 열기**로 `xapk` 로드
2. 경고 팝업은 무시 가능 (로딩 완료까지 대기)
3. 좌측 트리 주요 항목 확인:
   - `Inputs`
   - `소스코드`
   - `리소스`
   - `APK signature`
   - `Summary`
4. 패키지 기준 탐색:
   - `com` → `kakao` → `talk`

---

## 3. 로그인 관련 도메인/엔드포인트 추적
## 3-1. 도메인 검색
- 통합 검색(상단 돋보기)에서 `katalk.kakao.com` 검색
- `정의 검색`에서 `필드`, `코드` 활성화
- 결과가 10개만 보이면 **모두 로드**로 전체 조회

검색 중 `str = "katalk.kakao.com";`를 기준으로 상위 코드를 따라가면  
`com.kakao.talk.core.net.HostConfig` 및 관련 클래스 연결 확인 가능.

---

## 3-2. 클래스 추적 포인트
- `SubDeviceLogin`
- `AccountRegisterationService`
- `CreateAccountService`

`SubDeviceLogin` 경로에서 `login.json` 확인 가능:

```text
https://katalk.kakao.com/android/account/login.json
```

---

## 4. `SubDeviceLoginResponse` 기준 응답 필드
확인된 주요 응답 값(관찰 기준):
- `server_time`
- `userId`
- `profileId`
- `countryIso`
- `accountId`
- `access_token`
- `refresh_token`
- `token_type`
- `autoLogin` / `displayAccountId`
- `mainDeviceAppVersion`
- (과거 잔해) `sessionKey`

---

## 5. `login.json` 요청 파라미터 추적
`SubDeviceLoginParams` 기준으로 확인된 key:
- `email`
- `password`
- `device_uuid`
- `device_name`
- `auto_login`
- `autowithlock`
- `forced`
- `permanent`
- `passcode`
- `model_name`
- `another_email_verification_uri`

`Params` 네이밍상 기본 `Content-Type`은 다음으로 추정 가능:
```http
application/x-www-form-urlencoded
```

필수성 높게 보인 값(관찰 기준):
- `email`
- `password`
- `device_*`
- `forced`
- `permanent`
- `model_name`

---

## 6. 헤더 구성 포인트
## 6-1. User-Agent 계열
분석 중 확인된 문자열:
- `A`: `android/26.1.2/ko`
- `User-Agent`: `KT/26.1.2 An/13 ko`

## 6-2. `X-Vc` 생성 규칙
`XvcHeader` 추적 기준:
```text
X-Vc = sha512("BARD|" + userAgent + "|DANTE|" + duuid + "|SIAN").substring(0,16)
```

## 6-3. 기본 요청 헤더 예시
```http
A: android/26.1.2/ko
User-Agent: KT/26.1.2 An/13 ko
X-Vc: <sha512(...).substring(0,16)>
Content-Type: application/x-www-form-urlencoded
```

---

## 7. `status: -100` 해석
`login.json` 요청 시 `{ status: -100 }`이 오더라도  
즉시 실패라기보다 **기기 미등록 상태**로 해석

---

## 8. Passcode 로그인 흐름
## 8-1. 기본 구조
`PasscodeLogin` / `PasscodeRequest` 기준:

- JSON 바디 사용 (`Content-Type: application/json`)
- 데이터 예시:
  - `email`
  - `password`
  - `device.uuid` (상황에 따라 `name`, `model`, `osVersion` 추가)

---

## 8-2. Generate
엔드포인트:
```text
https://katalk.kakao.com/android/account/passcodeLogin/generate
```

요청 데이터(관찰 기준):
```json
{
  "email": "...",
  "password": "...",
  "permanent": true,
  "device": {
    "name": "...",
    "uuid": "...",
    "model": "...",
    "osVersion": "..."
  }
}
```

특이 헤더:
- `authorization`: 16자리 hex 랜덤 문자열 (polling 구분용으로 보임)

성공 응답 예:
```json
{ "status": 0, "passcode": "...", "remainingSeconds": 123 }
```

---

## 8-3. Cancel
엔드포인트:
```text
https://katalk.kakao.com/android/account/passcodeLogin/cancel
```
- `generate` 때 사용한 동일 `authorization` 헤더 사용
- 만료 전 polling 종료 가능

---

## 8-4. Register Device (상태 확인/등록)
엔드포인트:
```text
https://katalk.kakao.com/android/account/passcodeLogin/registerDevice
```

응답 관찰 예:
- 등록 가능 상태: `{ "status": 0 }`
- 아직 타이밍 아님: `{ "status": -100, "nextRequestIntervalInSeconds": 3 }`
- 그 외: 실패 케이스

---

## 9. 최종 로그인 완료 조건
기기 등록에서 `status: 0` 이후 `login.json` 재호출 시  
로그인 성공이라면 응답에 아래 필드가 핵심적으로 포함됨:
- `userId`
- `access_token`
- `refresh_token`

---

## 10. 요약 흐름도
1. `login.json` 시도
2. `status -100`이면 Passcode 플로우 진입
3. `passcodeLogin/generate`
4. 필요 시 `passcodeLogin/cancel` 또는 `registerDevice` polling
5. 등록 완료(`status 0`) 후 `login.json` 재요청
6. 토큰 수신 확인 (`access_token`, `refresh_token`)
