# Java Basic
---
## 클래스와 객체

### 클래스 선언
- **클래스(Class)**: 객체를 만들기 위한 설계도(템플릿)
- **객체(Object)**: 클래스로부터 생성된 실체 (인스턴스)
- 선언 방법: `public class 클래스명 { ... }`
- 객체 생성: `클래스명 변수명 = new 클래스명();`

### 참조 타입 (Reference Type)
- 기본 타입(int, double 등)과 달리 **메모리 주소(참조값)** 를 저장
- 참조 타입 종류: 클래스, 배열, 인터페이스
- `null`: 참조하는 객체가 없음을 의미
- 기본 타입은 Stack, 참조 타입 객체는 **Heap** 에 저장

### String 클래스
- 문자열을 다루는 대표적인 참조 타입
- 리터럴(`"hello"`) vs `new String("hello")` → 동일 내용이라도 `==` 비교 시 다를 수 있음
- 문자열 동등 비교는 반드시 **`.equals()`** 사용
```java
String s1 = "hello";
String s2 = new String("hello");
System.out.println(s1 == s2);       // false (주소 비교)
System.out.println(s1.equals(s2));  // true  (내용 비교)
```

### 필드(Field) 선언
- 클래스 내부에 선언된 변수 (객체의 속성/상태 표현)
- 선언 위치: 클래스 블록 `{ }` 내부, 메소드 외부
- 초기화 없으면 기본값 자동 부여 (int → 0, boolean → false, 참조타입 → null)
```java
public class Car {
    String color;   // 필드
    int speed;      // 필드
}
```

### 메소드(Method)
- 객체의 **동작(행동)** 을 정의하는 코드 블록
- 선언: `반환타입 메소드명(매개변수) { ... }`
- 반환값이 없으면 `void` 사용
- 호출: `객체.메소드명(인자);`
```java
public int add(int a, int b) {
    return a + b;
}
```

### String 클래스의 주요 메소드

| 메소드 | 설명 |
|---|---|
| `length()` | 문자열 길이 반환 |
| `charAt(index)` | 특정 인덱스 문자 반환 |
| `substring(start, end)` | 문자열 일부 추출 |
| `equals(str)` | 문자열 내용 비교 |
| `contains(str)` | 포함 여부 확인 |
| `replace(old, new)` | 문자열 치환 |
| `trim()` | 앞뒤 공백 제거 |
| `split(regex)` | 문자열 분리 |
| `toUpperCase()` | 대문자 변환 |
| `toLowerCase()` | 소문자 변환 |

### 변수의 Scope와 static
- **지역 변수**: 메소드 내부 선언, 메소드 종료 시 소멸
- **인스턴스 변수(필드)**: 객체마다 독립적으로 존재
- **static 변수(클래스 변수)**: 모든 객체가 **공유**, 클래스명으로 접근
- **static 메소드**: 객체 생성 없이 호출 가능 (`클래스명.메소드()`)
```java
public class Counter {
    static int count = 0;  // 모든 객체가 공유
    int id;                // 객체마다 개별 보유
}
```

### 열거형(enum)
- 관련된 상수들을 묶어 **타입 안전하게** 관리
- 선언: `enum 이름 { 상수1, 상수2, ... }`
```java
enum Season { SPRING, SUMMER, FALL, WINTER }
Season s = Season.SUMMER;
```

## 클래스 다듬기

### 생성자(Constructor)
- 객체 생성 시 **자동으로 호출**되는 특수 메소드
- 클래스명과 동일, 반환 타입 없음
- 기본 생성자: 명시하지 않으면 컴파일러가 자동 추가
```java
public class Car {
    String color;
    public Car(String color) {   // 생성자
        this.color = color;
    }
}
```

### this
- 현재 **객체 자신**을 참조하는 키워드
- 필드와 매개변수 이름이 같을 때 구분하기 위해 사용
- `this()`: 같은 클래스의 다른 생성자 호출
```java
public Car(String color) {
    this.color = color;  // this.color = 필드, color = 매개변수
}
```

### 메소드 오버로딩(Overloading)
- **같은 이름**의 메소드를 매개변수를 달리하여 여러 개 정의
- 매개변수의 타입, 개수, 순서가 달라야 함 (반환 타입만 다른 건 불가)
```java
int add(int a, int b) { ... }
double add(double a, double b) { ... }  // 오버로딩
```

### 생성자 오버로딩과 this()
- 생성자도 오버로딩 가능 → 다양한 방식으로 객체 초기화
- `this()` 로 다른 생성자를 호출하여 코드 중복 방지
```java
public Car() {
    this("흰색");         // 아래 생성자 호출
}
public Car(String color) {
    this.color = color;
}
```

### 패키지(Package)
- 클래스들을 **폴더처럼** 묶어 관리
- 선언: 파일 최상단에 `package 패키지명;`
- 다른 패키지 사용: `import 패키지명.클래스명;`

## 상속

### 상속(Inheritance)
- 부모 클래스의 필드와 메소드를 자식 클래스가 **물려받음**
- `extends` 키워드 사용, Java는 **단일 상속**만 지원
```java
public class Animal { ... }
public class Dog extends Animal { ... }  // Animal 상속
```

### 접근 제한자(Access Modifier)

| 제한자 | 같은 클래스 | 같은 패키지 | 자식 클래스 | 전체 |
|---|:---:|:---:|:---:|:---:|
| `public` | O | O | O | O |
| `protected` | O | O | O | X |
| `default` | O | O | X | X |
| `private` | O | X | X | X |

### 추상 클래스(Abstract Class)
- 직접 객체 생성 불가, **상속 전용** 클래스
- `abstract` 키워드, 추상 메소드(body 없음) 포함 가능
- 자식 클래스는 추상 메소드를 **반드시 오버라이딩** 해야 함
```java
public abstract class Shape {
    abstract double area();  // 추상 메소드
}
```

### super와 부모 생성자
- `super`: 부모 클래스를 참조
- `super()`: 부모 생성자 호출 (자식 생성자 첫 줄에 위치)
- 명시하지 않으면 컴파일러가 자동으로 `super()` 추가
```java
public Dog(String name) {
    super(name);  // 부모 Animal(name) 생성자 호출
}
```

### 오버라이딩(Overriding)
- 부모의 메소드를 자식에서 **재정의**
- 메소드명, 매개변수, 반환타입 동일해야 함
- `@Override` 어노테이션으로 명시 권장
```java
@Override
public void sound() {
    System.out.println("멍멍!");
}
```

### 클래스 형변환
- **업캐스팅(Upcasting)**: 자식 → 부모 타입, 자동 변환
- **다운캐스팅(Downcasting)**: 부모 → 자식 타입, 명시적 변환 필요
- `instanceof`: 객체 타입 확인
```java
Animal a = new Dog();          // 업캐스팅 (자동)
Dog d = (Dog) a;               // 다운캐스팅 (명시)
if (a instanceof Dog) { ... }  // 타입 확인
```

## 인터페이스와 다른 형식의 클래스

### 인터페이스(Interface)
- 클래스가 구현해야 할 **규약(명세)** 정의
- 모든 메소드는 기본적으로 `public abstract`
- 모든 변수는 `public static final` (상수)
- `implements` 키워드로 구현, **다중 구현** 가능
```java
public interface Flyable {
    void fly();  // 추상 메소드
}

public class Bird implements Flyable {
    @Override
    public void fly() { System.out.println("날아가요!"); }
}
```

### 인터페이스의 default method (Java 8+)
- 인터페이스 내에 **구현 코드를 포함**하는 메소드
- 기존 인터페이스에 메소드 추가 시 하위 호환성 유지 가능
- `default` 키워드 사용, 오버라이딩 가능
```java
public interface Flyable {
    default void land() {
        System.out.println("착륙합니다.");
    }
}
```

### 내부 클래스(Inner Class)
- 클래스 내부에 선언된 클래스
- 종류: 인스턴스 내부 클래스, 정적(static) 내부 클래스, 지역 클래스

### 익명 클래스(Anonymous Class)
- 이름 없이 **일회성**으로 클래스를 정의하고 즉시 생성
- 인터페이스나 추상 클래스를 즉석에서 구현할 때 사용
```java
Flyable f = new Flyable() {
    @Override
    public void fly() {
        System.out.println("익명 클래스로 날기!");
    }
};
f.fly();
```

## 예외처리

### Exception (예외)
- 프로그램 실행 중 발생하는 **비정상적인 상황**
- **Checked Exception**: 컴파일 시점에 처리 강제 (IOException 등)
- **Unchecked Exception**: 런타임에 발생 (NullPointerException, ArrayIndexOutOfBoundsException 등)
- `try-catch-finally` 구조로 처리
```java
try {
    // 예외 발생 가능 코드
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("0으로 나눌 수 없습니다: " + e.getMessage());
} finally {
    System.out.println("항상 실행");  // 선택사항
}
```

### throws
- 메소드에서 예외를 **직접 처리하지 않고 호출부로 전가**
- 메소드 선언부에 `throws 예외클래스` 명시
```java
public void readFile() throws IOException {
    // 예외를 처리하지 않고 호출자에게 넘김
}
```

### Exception 발생시키기 (throw)
- 의도적으로 예외를 발생시킬 때 `throw` 키워드 사용
```java
if (age < 0) {
    throw new IllegalArgumentException("나이는 음수일 수 없습니다.");
}
```

### 사용자 정의 Exception
- `Exception` 또는 `RuntimeException` 을 상속하여 커스텀 예외 생성
- 비즈니스 로직에 맞는 명확한 예외 표현 가능
```java
public class InsufficientBalanceException extends Exception {
    public InsufficientBalanceException(String message) {
        super(message);
    }
}

// 사용
throw new InsufficientBalanceException("잔액이 부족합니다.");
```

## 핵심 개념 요약

| 개념 | 키워드 | 핵심 포인트 |
|---|---|---|
| 상속 | `extends` | 코드 재사용, 단일 상속 |
| 인터페이스 | `implements` | 다중 구현, 규약 정의 |
| 추상 클래스 | `abstract` | 중간 단계, 객체 생성 불가 |
| 오버라이딩 | `@Override` | 부모 메소드 재정의 |
| 오버로딩 | 같은 이름 | 매개변수 다름 |
| 생성자 | 클래스명과 동일 | 객체 초기화 |
| 예외처리 | `try-catch-finally` | 비정상 상황 처리 |
| 형변환 | `instanceof` | 업/다운 캐스팅 |
