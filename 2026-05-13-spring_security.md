# Spring Security 인증과 인가 구현
Spring Security를 사용해서 인증(Authentication)과 인가(Authorization)를 구현

- 커스텀 로그인 페이지
- 사용자 로그인 처리
- 사용자 권한별 접근 제어
- 접근 거부 페이지 처리
- 메서드 레벨 권한 검사
- 간단한 테스트 코드 작성

## 인증과 인가의 차이
### 인증(Authentication)
인증은 "이 사용자가 누구인가?"를 확인하는 과정

즉, 로그인 과정에서 아이디와 비밀번호를 검사해서 사용자를 확인하는 것이 인증

### 인가(Authorization)
인가는 "이 사용자가 이 기능에 접근할 수 있는가?"를 확인하는 과정

즉, 로그인한 이후에도 사용자 권한에 따라 접근 가능한 페이지를 나누는 것이 인가

정리하면,

- 인증: 사용자를 확인하는 것
- 인가: 사용자의 권한을 확인하는 것

## 프로젝트 구조
역할을 나눠서 클래스를 구성

- `SecurityConfig`
  - Spring Security 전체 설정 담당
- `HomeController`
  - 페이지 이동 처리
- `SecureMessageService`
  - 권한에 따라 접근 가능한 메서드 제공

## 구현 과정
## 1. SecurityConfig 작성
가장 중요한 부분은 `SecurityConfig`!
여기서 어떤 URL을 누구에게 허용할지, 로그인은 어떻게 처리할지, 로그아웃은 어디로 이동할지 등을 설정

- `/`, `/login`, `/denied`, `/css/**` 는 모두 접근 가능
- `/user/**` 는 `USER` 또는 `ADMIN` 권한 필요
- `/admin/**` 는 `ADMIN` 권한 필요
- 그 외 요청은 로그인 필요
- 로그인 페이지는 `/login`
- 로그아웃 후 메인 페이지로 이동
- 권한이 없으면 `/denied` 페이지로 이동

이 설정을 통해 URL 단위 인가를 구현했다.

## 2. 사용자 정보 등록
이번 과제에서는 DB 연동 대신 메모리 기반 사용자 정보를 사용
`UserDetailsService`를 Bean으로 등록해서 테스트 계정을 제작

### 테스트 계정
- `user / 1234`
  - 권한: `ROLE_USER`
- `admin / 1234`
  - 권한: `ROLE_USER`, `ROLE_ADMIN`

비밀번호는 평문 그대로 사용하지 않고 `BCryptPasswordEncoder`로 암호화함
Spring Security에서는 비밀번호를 안전하게 관리하기 위해 인코더를 사용하는 것이 중요함

## 3. 로그인 페이지 구성
Thymeleaf 템플릿으로 `/login` 화면을 구성했고, 로그인 실패 시 에러 메시지, 로그아웃 후 성공 메시지가 보이도록 처리

Spring Security가 단순히 보안 설정만 하는 것이 아니라, 실제 화면과도 자연스럽게 연결됨

## 4. 사용자 페이지와 관리자 페이지 분리
권한 차이를 눈으로 확인하기 위해 `/user` 페이지와 `/admin` 페이지를 따로 제작

- `user` 계정으로 로그인하면 `/user`는 접근 가능
- `user` 계정으로 `/admin` 접근 시 거부
- `admin` 계정으로 로그인하면 `/user`, `/admin` 모두 접근 가능

## 5. 메서드 레벨 보안 적용
URL 단위 보안만 적용한 것이 아니라, 서비스 메서드에도 권한 검사를 추가
`@EnableMethodSecurity`를 활성화한 뒤 `@PreAuthorize`를 사용

- 사용자 메시지를 반환하는 메서드: `USER` 또는 `ADMIN` 권한 필요
- 관리자 메시지를 반환하는 메서드: `ADMIN` 권한 필요

이렇게 하면 컨트롤러에서 접근을 막는 것 외에도, 서비스 계층 자체를 보호 가능

## 6. 접근 거부 처리
로그인은 했지만 권한이 부족한 경우에는 `/denied` 페이지로 이동하도록 설정

인증 실패와 인가 실패가 다름

- 로그인하지 않음: 로그인 페이지로 이동
- 로그인했지만 권한 없음: 접근 거부 페이지로 이동

## 7. 테스트 코드 작성
MockMvc를 사용해서 간단한 보안 테스트 작성

- 메인 페이지는 누구나 접근 가능한지 확인
- 로그인하지 않은 사용자가 보호된 페이지 접근 시 로그인 페이지로 이동하는지 확인
- `USER` 권한 사용자가 `/user`에 접근 가능한지 확인
- `USER` 권한 사용자가 `/admin`에 접근하면 거부되는지 확인
- `ADMIN` 권한 사용자가 `/admin`에 접근 가능한지 확인
