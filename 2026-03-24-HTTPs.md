# HTTP / HTTPS
> 보안 -> 암호화 유무 차이

## HTTP
암호화가 되지 않은 평문을 주고 받음

- 데이터가 그대로 보임
- 중간에서 보면 그대로 드러남

## HTTPS
SSL/TLS 를 사용해 암호화화여 주고 받음

- 중간에서 보아도 해석이 어려움
- 안전한 통신 가능

||HTTP|HTTPS|
|:-:|:-:|:-:|
|secure|X|O|
|data|plain|encrypted|
|port|80|443|


# HTTP Methods
- GET - 리소스 조회 ( URL 입력, 링크 클릭 등 )
- POST - 새로운 리소스 생성 ( 성공적으로 생성 시 201 status )
- PUT - 기존 리소스 수정, 없으면 새로 생성 ( 덮어쓰기 )
- DELETE - 리소스 삭제
- PATCH - 리소스의 일부분 수정 ( PUT 은 전체 수정 )
- HEAD - GET 과 유사하나, body 를 제외하고 조회
- OPTIONS - 서버 <-> 브라우저 통신을 위한 옵션 확인 ( 서버의 method, header, content-type 확인 )
- CONNECT - 대상 자원으로 식별된 서버에 대한 연결 요청

# HTTP의 무상태성 (Stateless)
> HTTP 는 무상태 프로토콜이다.
> 
> 서버가 클라이언트의 이전 요청 상태를 기억하지 않는다.
>
> 매 요청이 전부 독립적이라는 뜻이다.

## 특징
> 1. 각 요청은 독립적:
이전 요청과 관계 없음
> 2. 서버는 상태를 저장하지 않음:
사용자 정보 유지 X
> 3. 요청마다 필요한 정보 포함:
매번 인증 정보 필요

## 장점
> 1. 확장성:
서버를 늘리기 쉬움
> 2. 서버 단순화:
상태 관리 로직 필요 없음
> 3. 장애 대응 유리:
특정 서버 죽어도 다른 서버가 처리 가능

## 단점
> 1. 매 요청마다 정보 전송 필요:
로그인 정보 계속 보내야 함
> 2. 상태 유지 어려움:
사용자 경험 구현이 번거로움
