# 객체지향 (OOP) 개념 4가지

### 추상화
복잡한 내부 구현을 숨기고 필요한 기능만 외부에 제공하는 것

무언가를 하는가에 집중하며 공통적인 속성과 기능을 제공

### 캡슐화
데이터와 메소드를 하나로 묶고 외부에서 접근하지 못하도록 제한하는 것

데이터를 보호하며 접근 제한자인 `private, protected` 등을 사용

### 상속
기존 클래스의 속성과 기능을 물려받아 새로운 클래스를 만드는 것

코드를 재사용하며 계층 구조를 형성

- 계층 구조 - 개체나 항목들이 위아래 또는 동등한 수준으로 나열된 형태

### 다형성
같은 이름의 메소드가 상황에 따라 다르게 작동하는 것

- 오버라이딩 - 부모 클래스에서 정의된 메소드를 자식 클래스에서 재정의하는 것

# HTTP / HTTPS
> 보안 -> 암호화 유무 차이

## HTTP
암호화가 되지 않은 평문을 주고 받음

- 데이터가 그대로 보임
- 중간에서 보면 그대로 드러남

## HTTPS
SSL/TLS 를 사용해 암호화화여 주고 받음

- 중간에서 보아도 해석이 어려움
- 안전한 통신 가능

||HTTP|HTTPS|
|:-:|:-:|:-:|
|secure|X|O|
|data|plain|encrypted|
|port|80|443|
