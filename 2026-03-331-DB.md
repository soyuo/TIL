# 1. DB(Database)란?

*DB(Database)*는
**데이터를 저장하고, 관리하고, 필요할 때 꺼내 쓰기 위한 시스템**이다

---

# 기본 개념

* **데이터(Data)** → 저장되는 정보 (이름, 나이, 주문 등)

* **테이블(Table)** → 데이터를 표 형태로 저장

* **행(Row)** → 하나의 데이터 (ex. 사용자 1명)

* **열(Column)** → 데이터 속성 (이름, 이메일 등)

---

# 그래서 왜 쓰이는가?

### 1) 데이터 영구 저장

* 서버가 꺼져도 데이터가 유지된다

### 2) 빠른 조회

* 검색, 정렬, 필터등의 기능을 지원한다

### 3) 데이터 관리

* 수정 / 삭제 / 추가를 체계적으로 처리한다

### 4) 동시성 처리

* 여러 사용자가 동시에 접근이 가능하다

---

# 3. DB의 종류

## 1) 관계형 데이터베이스 (RDB)

* 테이블 구조를 사용한다
* SQL 언어를 사용한다

* MySQL
* PostgreSQL
* Oracle Database

---

## 2) NoSQL 데이터베이스

* 자유로운 구조다 (JSON 형태 등)

* MongoDB (문서형)
* Redis (키-값)
* Cassandra (분산형)

---

# SQL (Structured Query Language)

→ 관계형 DB 에서 쓰이는 표준 프로그래밍 언어

SQL은 기능에 따라 크게 3가지로 나뉜다

---

## 1. DDL (Data Definition Language)

→ **데이터 구조를 정의하는 언어**

**테이블이나 DB 자체를 생성/수정/삭제**할 때 사용한다

### 주요 명령어

* **CREATE** → 생성
* **ALTER** → 수정
* **DROP** → 삭제
* **TRUNCATE** → 데이터 전체 삭제 (빠름)

### 예시

```
CREATE TABLE users (
  id INT,
  name VARCHAR(50)
);

ALTER TABLE users ADD age INT;

DROP TABLE users;
```

### 특징

* 데이터가 아니라 **구조(틀)**를 다룬다
* 실행하면 자동으로 반영된다 (Rollback 어려움)

---

## 2. DML (Data Manipulation Language)

→ **데이터를 실제로 조작하는 언어**

**데이터를 넣고, 조회하고, 수정하고, 삭제**한다

### 주요 명령어

* **SELECT** → 조회
* **INSERT** → 생성
* **UPDATE** → 수정
* **DELETE** → 삭제

### 예시

```
SELECT * FROM users;

INSERT INTO users (name, age)
VALUES ('홍길동', 20);

UPDATE users
SET age = 21
WHERE id = 1;

DELETE FROM users
WHERE id = 1;
```

### 특징

* 가장 많이 사용되는 SQL이다
* 트랜잭션과 함께 사용된다 (Rollback 가능)

---

## 3. DCL (Data Control Language)

→ **데이터 접근 권한을 제어하는 언어**

**누가 DB를 사용할 수 있는지 관리**한다

### 주요 명령어

* **GRANT** → 권한 부여
* **REVOKE** → 권한 회수

### 예시

```
GRANT SELECT ON users TO user1;

REVOKE SELECT ON users FROM user1;
```

### 특징

* 사용자별 접근 권한 설정이 가능하다

---

## 번외. TCL (Transaction Control Language)

→ **트랜잭션을 제어하는 언어**

### 주요 명령어

* **COMMIT** → 작업 확정
* **ROLLBACK** → 작업 취소

---

# 5. 핵심 특징

### 1) 무결성 (Integrity)

* 데이터 정확성을 유지한다

### 2) 독립성 (Independence)

* 구조가 바뀌어도 프로그램 영향이 적다

### 3) 트랜잭션 (Transaction)

* 작업 단위가 보장된다 (All or Nothing)

---

# 6. 장점

### 1) 중복 최소화

* 같은 데이터 반복 저장이 방지된다

### 2) 빠른 처리 속도

* 인덱스 등을 활용하여 빠르게 처리할 수 있다

### 3) 안정성

* 장애 발생 시 복구가 가능하다

* Rollback

---

# 7. 단점

### 1) 초기 설계 중요

* 잘못 설계하면 수정이 어렵다

### 2) 비용 발생

* 서버 + 유지 비용이 발생한다

### 3) 복잡성

* 대규모 시스템일수록 관리가 어렵다

---

# 요약

* DB는 **데이터를 저장하고 관리하며, 필요할 때 빠르게 조회·수정·삭제할 수 있도록 도와주는 시스템**
