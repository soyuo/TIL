# Spring MVC 패턴
## MVC 패턴
**MVC(Model-View-Controller)** 는 애플리케이션을 역할별로 나눠 **유지보수성과 확장성을 높이기 위한 구조**

---

## ● M: Model
* 애플리케이션의 **데이터와 상태를 관리하는 영역**
* DB 조회 결과나 비즈니스 로직의 결과 데이터를 담음
* Controller에서 View로 전달되는 데이터 컨테이너 역할

### 특징
* 단순 데이터 객체 (DTO, Entity 등)
* View와 Controller에 의존하지 않음

---

## V: View
* 사용자에게 보여지는 **UI(화면)** 담당
* Model 데이터를 기반으로 HTML 등을 생성
* Spring에서 주로 사용

* Thymeleaf
* JSP

### 특징
* 로직은 최소화 (표현만 담당)
* Model 데이터를 출력하는 역할

---

## C: Controller
* 클라이언트 요청을 받아 처리하는 **진입점**
* Service를 호출하여 비즈니스 로직 실행
* 결과를 Model에 담고 View 선택

### 흐름
1. 요청 받음
2. Service 호출
3. 결과 Model에 저장
4. View 반환

---

## DispatcherServlet
* Spring MVC의 **프론트 컨트롤러**
* 모든 요청이 가장 먼저 거치는 관문

### 역할
1. 요청 수신
2. Controller 검색 (Handler Mapping)
3. 실행 및 결과 반환
4. View Resolver 를 통해 View 검색
5. 최종 응답 반환

# Spring 게시판에 쓰인 기술
## JPA (Java Persistence API)
* Java 객체와 DB 테이블을 매핑하는 **ORM 기술**

### 특징
* SQL 없이 메서드로 DB 처리
* 객체 중심 개발 가능
* 생산성 증가

### 예시
```java
userRepository.findById(1L);
```

---

## Thymeleaf
* 서버 사이드 템플릿 엔진

### 특징
* HTML 기반 (디자인과 코드 분리 용이)
* Spring과 자연스럽게 통합
* `${}` 문법으로 데이터 출력

### 예시
```html
<p th:text="${user.name}"></p>
```

---

## MySQL
* 대표적인 관계형 데이터베이스(RDBMS)

### 특징
* 데이터 무결성
* 트랜잭션 지원
* 빠른 조회 가능

# Spring 주요 개념
## Entity
* DB 테이블과 매핑되는 **객체**

### 특징
* `@Entity` 어노테이션 사용
* 테이블의 한 행(row)을 표현

### 예시
```java
@Entity
public class User {
    @Id
    private Long id;
    private String name;
}
```

### 핵심
* DB 구조와 직접 연결됨

---

## Controller
* 클라이언트 요청을 처리하는 계층

### 특징
* `@Controller`, `@RestController` 사용
* URL과 메서드 매핑

### 예시
```java
@GetMapping("/users")
public String getUsers(Model model) {
    model.addAttribute("users", userService.findAll());
    return "userList";
}
```

---

## Service
* **로직 처리 계층**

### 역할
* 데이터 가공
* 트랜잭션 처리
* 여러 Repository 조합

### 특징
* Controller와 Repository 사이 중간 계층

### 예시
```java
public List<User> findAll() {
    return userRepository.findAll();
}
```

---

## Repository
* DB 접근을 담당하는 계층

### 특징
* 인터페이스로 선언
* Spring Data JPA가 구현 자동 생성

### 예시
```java
public interface UserRepository extends JpaRepository<User, Long> {
}
```

### 핵심
* SQL 없이 DB 접근 가능

---

## DTO (Data Transfer Object)
* 계층 간 데이터 전달을 위한 객체
* Entity를 그대로 노출하면 위험하기에 사용
*   (보안 문제 / 구조 의존성 증가)

### 특징
* 필요한 데이터만 포함
* 요청/응답 전용 객체

### 예시
```java
public class UserDTO {
    private String name;
}
```

### 핵심
* **데이터 보호 + 구조 분리**

# 전체 흐름
Spring 게시판 요청 흐름:

```
클라이언트 요청
   ↓
DispatcherServlet
   ↓
Controller
   ↓
Service
   ↓
Repository
   ↓
DB (MySQL)
   ↓
결과 반환
   ↓
Service
   ↓
Controller
   ↓
Model
   ↓
View (Thymeleaf)
   ↓
사용자 화면 출력
```

---

## 한 줄 핵심 정리
* MVC: 역할 분리 구조
* Controller: 요청 처리
* Service: 로직 처리
* Repository: DB 접근
* Entity: DB 데이터 구조
* DTO: 데이터 전달용 객체
* JPA: 객체 기반 DB 처리
* Thymeleaf: 화면 출력
